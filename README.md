# Раскраска по номерам — оффлайн

Простое локальное веб-приложение, которое превращает загруженное изображение в шаблон «раскраски по номерам». Всё работает в браузере через Canvas API, без отправки файлов на сервер и без нейросетей.

## Установка

Зависимости не требуются. Нужен только браузер и любой локальный статический сервер.

```bash
npm install
```

Команда не скачивает пакеты, потому что проект не использует внешние npm-зависимости.

## Запуск

```bash
npm start
```

Затем откройте:

```text
http://localhost:4173
```

Альтернатива без npm:

```bash
python3 -m http.server 4173
```

## Как пользоваться

1. Нажмите «Загрузить изображение» и выберите PNG/JPEG/WebP.
2. При необходимости настройте:
   - количество цветов: от 8 до 30, по умолчанию 18;
   - минимальный размер области;
   - силу предварительного сглаживания;
   - плавность контуров;
   - толщину линий;
   - размер цифр.
3. Нажмите «Создать раскраску».
4. Проверьте три предпросмотра:
   - исходник;
   - цветовая карта;
   - финальная раскраска с белым фоном, контурами и номерами.
5. Скачайте результат кнопками «Скачать PNG» или «Скачать SVG».

## Используемые алгоритмы

Приложение использует классический математический pipeline обработки изображений:

1. **Canvas API loading/downscale** — изображение загружается локально и уменьшается до 1200 px по длинной стороне, чтобы сохранить интерактивную скорость.
2. **Separable box blur** — предварительно сглаживает шум и мелкие текстуры. В первой версии выбран вместо bilateral filter, потому что он стабилен, быстрый и не требует зависимостей.
3. **K-Means по RGB** — разбивает изображение на заданное число основных цветов. Центры сортируются по яркости, поэтому номера палитры стабильнее воспринимаются визуально.
4. **Majority smoothing** — локально убирает единичные шумовые пиксели на карте цветов.
5. **Connected components** — находит связные области одинакового цвета.
6. **Удаление мелких компонентов** — маленькие области присоединяются к соседней области с учетом длины общей границы и похожести цвета.
7. **Grid contour tracing** — строит контуры по границам связных областей.
8. **Ramer–Douglas–Peucker** — упрощает контуры, убирая лишнюю пиксельную детализацию.
9. **Chaikin smoothing** — сглаживает линии, чтобы избежать квадратного «Minecraft»-эффекта и сохранить узнаваемые формы.
10. **Distance-transform visual center** — номер ставится не в центр bounding box, а в самую «глубокую» внутреннюю точку области. Это проще и быстрее классического polylabel для большого числа растровых компонентов и гарантирует, что цифра остается внутри области.
11. **PNG/SVG export** — PNG берется с финального Canvas, SVG собирается из тех же сглаженных контуров и текстовых номеров.

## Ограничения первой версии

- K-Means работает в RGB, а не в Lab. Для большинства изображений это быстрее и достаточно надежно, но perceptual clustering можно улучшить.
- Контуры строятся по растровым границам, затем упрощаются и сглаживаются. Очень тонкие детали исходника могут исчезать при большом минимальном размере области или сильном сглаживании.
- Номера не ставятся в слишком маленькие или узкие области, чтобы не вылезать за границы.
- Для очень больших или очень детальных изображений генерация может занять несколько секунд, потому что всё считается в одном потоке браузера.
- SVG содержит только черные контуры и номера на белом фоне; заливка цветов намеренно не экспортируется, так как это шаблон для раскрашивания.

## Что можно улучшить дальше

- Перевести K-Means в Lab или Oklab для более визуально равномерной палитры.
- Добавить Web Worker, чтобы интерфейс не блокировался во время обработки больших фото.
- Реализовать полноценный marching squares/polylabel для более точных векторных контуров и центров сложных многоугольников.
- Добавить предпросмотр с заливками и отдельный экспорт цветной легенды.
- Добавить автоматический подбор минимального размера области по размеру изображения.
- Добавить режим печати в PDF.
